## Getting Access

Pucked uses an **invitation-based system** to ensure quality and security. To get access:

1. **Request an invitation** from your administrator
2. **Check your email** for the invitation code
3. **Follow the signup process** outlined below

## Signing Up

### Step 1: Login with GitHub

1. Visit the application URL
2. Click the **"Login with GitHub"** button
3. Authorize the application with your GitHub account

### Step 2: Enter Invitation Code

After logging in with GitHub, you'll be prompted to enter an invitation code:

1. **Copy the invitation code** from your email
2. **Paste it** into the invitation code field
3. **Click "Submit"** to activate your account

### Step 3: Access the Dashboard

Once your invitation is accepted, you'll be redirected to the admin dashboard where you can start creating content.

## Dashboard Overview

The admin dashboard is your control center for managing content. Here's what you'll find:

### Navigation

- **Left Sidebar**: Page tree and structure
- **Main Area**: Page editor and preview
- **Right Sidebar**: Page properties and translations

### Key Features

#### 1. Page Management

- **Create Pages**: Add new pages to your site
- **Edit Pages**: Modify existing pages visually
- **Organize**: Arrange pages in a hierarchical structure
- **Publish**: Make pages live on your site

#### 2. Visual Editor

The visual editor lets you build pages without coding:

- **Drag and Drop**: Move components to rearrange
- **Edit Content**: Click and edit text directly
- **Customize**: Adjust styles and properties
- **Preview**: See changes in real-time

#### 3. Bilingual Support

Create content in multiple languages:

- **English**: Default language
- **Farsi**: Right-to-left language support
- **Switch Languages**: Toggle between languages easily

## Creating Your First Page

### Step 1: Create a New Page

1. Click the **"New Page"** button in the dashboard
2. Enter a **page title** (e.g., "About Us")
3. Choose a **parent page** (optional, for hierarchical structure)
4. Click **"Create"**

### Step 2: Add Content

1. **Select a component** from the component palette
2. **Drag it** onto your page
3. **Edit the content** by clicking on it
4. **Customize styles** using the properties panel

### Step 3: Configure Page Settings

In the right sidebar, you can:

- **Set the URL slug** for the page
- **Add meta description** for SEO
- **Configure language settings**
- **Set page visibility**

### Step 4: Preview and Publish

1. Click **"Preview"** to see how your page looks
2. Make any necessary adjustments
3. Click **"Publish"** to make your page live

## Tips for Success

### Content Best Practices

1. **Keep it Simple**: Start with basic layouts
2. **Use Headings**: Organize content with clear headings
3. **Add Descriptions**: Help search engines understand your pages
4. **Test Preview**: Always preview before publishing

### Design Tips

1. **Consistency**: Use similar styles across pages
2. **Whitespace**: Don't be afraid of empty space
3. **Mobile-Friendly**: Check how your page looks on mobile
4. **Accessibility**: Use descriptive text for links and images

### Bilingual Content

1. **Translate Key Pages**: Start with important pages
2. **Cultural Context**: Consider cultural differences
3. **RTL Support**: Farsi automatically uses right-to-left layout
4. **Consistent Navigation**: Keep navigation similar in both languages

## FAQ

### How do I delete a page?

Click the page in the tree, then click the "Delete" button. Note: You can't delete pages with child pages.

### Can I restore a deleted page?

Currently, deleted pages cannot be restored. Make sure to export important content before deletion.

### How do I change the page URL?

Edit the "Slug" field in the page properties panel. This will update the page's URL.

### What's the difference between draft and published?

- **Draft**: Only visible to editors
- **Published**: Visible to everyone on your site
